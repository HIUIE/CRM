# SmartTrade AI CRM — 部署与使用指南

> 适用于 2-5 人小团队单机或局域网共享使用

---

## 一、快速启动

### 前置要求

- **Node.js** ≥ 18（推荐 20 LTS）
- **npm** ≥ 9
- 操作系统：macOS / Linux / Windows

### 安装步骤

```bash
# 1. 进入项目目录
cd smarttrade-crm

# 2. 安装依赖
npm install

# 3. 复制环境配置
cp .env.example .env

# 4. 修改 .env 中的必要配置
#    JWT_SECRET — 务必改为一个长随机字符串（生产环境必须设置）
#    INITIAL_ADMIN_PASSWORD — 管理员初始密码，首次启动后记得修改

# 5. 构建前端
npm run build

# 6. 启动服务
npm run dev

# 7. 打开浏览器访问
#    http://localhost:3000
```

### 默认管理员账号

- 用户名：`root`
- 密码：`.env` 中 `INITIAL_ADMIN_PASSWORD` 设置的值（首次启动后请在设置中修改）

## 三、系统功能概览

### 业务控制台（Dashboard）

全局数据面板，展示：
- 订单总数、待收金额、运输中订单
- 今日待办事项（逾期收款、缺失报关、缺物流）
- 最近动态时间线
- 快捷操作入口
- 订单状态分布环形图

### 客户管理

- **客户列表**：搜索、国家筛选、分页
- **客户 360 画像**：客户详情页包含个人资料卡、跟进记录、关联订单、财务流水、任务、联系人、系统动态
- **客户跟进**：记录每一次沟通内容，支持按时间线查看

### 订单管理

核心业务模块，一条完整闭环：

1. **订单创建**：填写订单号、关联客户、产品摘要、总金额
2. **商品明细**：多产品行录入（名称、规格、数量、单价）
3. **生产安排**：关联工厂伙伴、生产状态跟踪（排产中→生产中→已完成）、质检状态
4. **生产日志**：记录每条生产进度节点
5. **财务流水**：收付款记录（多币种 USD/CNY）、核销状态、附件
6. **报关信息**：报关单号、贸易方式、申报/放行日期
7. **装箱记录**：件数、尺寸、毛重/净重、现场照片
8. **物流轨迹**：国内+国际段运输、承运商、单号、ETD/ETA
9. **核心单据凭证库**：上传所有业务凭证 PDF/图片
10. **订单跟进时间轴**：记录每次业务沟通
11. **关联任务**：指派内部待办

### 财务流水

- 多币种收付款记录（USD/CNY）
- 汇总统计卡片（收款/付款总额）
- 关联订单和伙伴

### 物流打包

- 物流记录管理（跟踪号、承运商、状态）
- 国内/国际段划分
- 分页与状态筛选

### 合作伙伴

- **伙伴列表**：搜索、类型筛选（工厂/货代/报关行）、星级评分
- **伙伴 360 画像**：详情页包含档案卡片、合作概览（本/上月订单数）、关联订单、财务流水

### 团队协同看板（Tasks）

- 三列看板：待处理 → 进行中 → 已完成
- 任务指派、优先级（P0/P1/P2）
- 任务评论与附件

### AI 助手

- 多模型支持：DeepSeek / OpenAI / Gemini
- 订单风险诊断
- 聊天记录本地持久化
- 可在系统设置中配置 API Key

### 系统设置

- **常规配置**：订单编号前缀、默认币种
- **数据导出**：Excel 工作簿（12 Sheet）/ 客户订单归档（含附件）/ CSV
- **团队管理**：创建/编辑/停用账号、重置密码
- **AI 配置**：提供商选择、模型选择、API Key、连接测试

### 审计日志

记录所有核心实体的增删改操作，支持查询与详情查看。

## 四、安全说明

### 已内置的安全措施

| 措施 | 说明 |
|------|------|
| JWT 认证 | 24h 过期 httpOnly cookie |
| 角色权限 | admin（管理员）/ staff（业务员）两级权限 |
| 密码加密 | bcrypt 10 轮哈希 |
| 敏感路径保护 | 阻止直接访问 .env/.git/.db 文件 |
| 附件安全 | UUID 重命名、路径 containment 检查 |
| 前端隐私盾 | 默认掩码显示邮箱/电话 |
| 登录限流 | 5 次/15 分钟 |
| CSRF 防护 | httpOnly cookie + sameSite 策略 |
| 审计日志 | 所有 C/U/D 操作记录，30 天自动清理 |
| 全局错误处理 | express-async-errors + 统一错误中间件 |

### 部署安全建议

1. **必须修改 JWT_SECRET**：生成一个长随机字符串
2. **使用 HTTPS**：生产环境务必配置反向代理（Nginx）提供 TLS
3. **修改默认 root 密码**：首次登录后立即修改
4. **定期备份**：可用 `npm run backup` 备份数据库和附件
5. **限制局域网访问**：`.env` 中设置 `HOST=127.0.0.1` 仅本地，或 `HOST=0.0.0.0` 开放局域网

## 五、数据导出

系统支持三种导出格式：

### 1. Excel 工作簿（推荐）
单个 `.xlsx` 文件，12 个 Sheet：
- 订单、商品明细、财务流水、物流、报关、生产、装箱
- 客户、合作伙伴、任务、客户跟进、订单跟进
- 带自动筛选和蓝色表头样式

### 2. 客户订单归档（ZIP）
按客户分目录的结构化归档：
```
客户/
├── 客户信息.xlsx        # 概览 + 联系人 + 跟进 + 订单列表
└── orders/
    ├── ORD-2026-0001/
    │   ├── 订单详情.xlsx  # 订单全部数据
    │   └── attachments/  # 附件原文件
    └── ORD-2026-0002/
        └── ...
```

### 3. CSV 表格导出
多个 CSV 文件的 ZIP 压缩包，适合 Excel 传统分析。

## 六、常见问题

### Q: 如何修改数据库位置？
修改 `.env` 中的 `CRM_DB_PATH`，指向新的 SQLite 文件路径。

### Q: 如何重置管理员密码？
1. 停止服务
2. 删除 `data/erp_database_v2.sqlite`
3. 设置 `.env` 中的 `INITIAL_ADMIN_PASSWORD`
4. 重启服务 — 系统将自动创建新数据库和 root 用户

### Q: 如何配置 AI 功能？
在 系统设置 → AI 配置 中：
1. 选择提供商（DeepSeek / OpenAI / Gemini）
2. 输入 API Key
3. 可选：自定义 Base URL（代理地址）
4. 点击"测试连接"验证
5. 保存

### Q: 附件存在哪里？
默认存储在 `data/uploads/` 目录，按客户 ID 组织子目录。

### Q: 如何备份？
```bash
npm run backup
```
备份文件保存在 `data/backups/` 目录。

- **前端**：React 19 + TypeScript + Tailwind CSS v4 + Vite
