# Changelog

## [2026-04-24]

### 生产上线加固
- 新增生产启动脚本 `npm run start`，以 `NODE_ENV=production` 启动并服务 `dist/` 静态资源。
- 新增一键上线检查 `npm run release:check`，依次执行：类型检查、测试、构建、UI smoke。
- 服务启动日志补齐生产排查信息：运行模式、本机/局域网访问提示、数据库路径、上传目录。
- 启动失败信息更明确（如端口占用、无权限监听端口等）。

### 局域网登录与安全初始化
- Cookie 策略改为局域网 HTTP 友好：默认不强制 `secure`，仅 `COOKIE_SECURE=true` 才启用 HTTPS-only Cookie。
- 生产环境强制要求配置 `JWT_SECRET`，避免继续使用预览默认密钥。
- 新库首次初始化管理员 `root`：生产环境必须设置 `INITIAL_ADMIN_PASSWORD`，否则启动会直接失败。

### 健康检查与路径可配置
- `/api/health` 扩展返回：`dbPath`、`uploadsPath`、`uploadsUrl`、`timestamp`，便于生产排查。
- 上传目录支持环境变量 `UPLOADS_DIR`，避免 smoke/测试数据写入正式附件目录。

### Smoke 与备份
- 新增 `npm run smoke:ui`：使用临时数据库与演示数据进行生产模式 smoke（有 Playwright 则跑浏览器，无则降级为 HTTP/API 检查）。
- 新增 `npm run backup`：将 SQLite 数据库与 `uploads/` 备份到按时间戳命名的目录（可用 `BACKUP_DIR` 指定位置）。

### 文档
- `.env.example` 更新为局域网生产可用模板，补齐 `HOST/PORT/CRM_DB_PATH/UPLOADS_DIR/JWT_SECRET/INITIAL_ADMIN_PASSWORD/COOKIE_SECURE`。
- README 补充“第一版生产部署步骤”、上线前检查、备份与恢复流程。

## [2026-04-23]

### 团队稳定化
- 后端路由继续按领域收口，统一保留标准错误结构：`error.code` + `error.message`。
- 新增基础用户管理接口，支持创建账号、启停用户、重置密码。
- 权限边界进一步收紧：
  - `admin` 管理系统设置、AI 配置、团队账号和关键删除动作
  - `staff` 只负责业务查看与编辑
- `/settings` 前端入口改为管理员专属路由，业务账号不再进入系统设置页。
- 登录页和系统设置页文案改为正式团队入口语气，不再展示默认管理员账号。

### 前端结构与一致性
- 新增共享类型层：
  - `src/types/api.ts`
  - `src/types/crm.ts`
- 客户、伙伴、订单、财务、物流等主要列表页开始复用统一类型，而不是各页面重复定义接口。
- 主要列表页补充“创建人 / 最近操作人”回显，方便团队协作场景下追踪记录来源。
- AI 页面文案收敛为“订单草稿解析 + 风险诊断”两类辅助用途，不再强调测试态限制。

### 测试与演示环境
- 新增无端口回归测试 `tests/core.test.ts`，覆盖：
  - 认证与权限
  - 用户管理关键逻辑
  - 订单详情聚合
  - 生产 / 财务 payload 校验
- 新增 `npm run seed:demo` 演示数据脚本：
  - 创建演示业务账号
  - 补齐客户、伙伴、订单、生产、财务、报关、物流示例数据
- README 重写为“局域网小团队可用”版本，新增：
  - `/api/health`
  - `CRM_DB_PATH`
  - `uploads/`
  - 局域网部署建议
  - SQLite 与附件目录备份说明

### AI
- AI 解析入口不再被硬编码限制为 Gemini。
- `/api/ai/parse-order` 现在会根据系统设置中的模型自动分流：
  - `Gemini` 继续走 Google GenAI SDK
  - `deepseek-chat` 改走 OpenAI 兼容接口
- 新增对 `ai_base_url` 的兼容读取；当模型为 DeepSeek 且未手填 Base URL 时，后端会优先使用默认官方兼容地址。
- AI 页面去掉“仅支持 Gemini”的前端限制提示，只要配置了可用 Key 即可发起解析。

### 订单工作台
- 订单详情页继续重构为高密度工作台，形成稳定的模块顺序：
  - 基础信息
  - 订单明细
  - 生产安排
  - 财务流转
  - 报关信息
  - 物流信息
- 顶部阶段条支持锚点跳转到对应模块。
- 顶部财务概览支持 `待收金额 = 0` 时显示 `已结清` 状态。
- 基础信息区域移除了 `关键里程碑` 与 `产品摘要` 的主展示。
- 新增 `生产安排` 模块与 Drawer，支持选择工厂/供应商、维护生产状态与验货状态。
- 报关模块支持“后补附件”，无需重新打开完整报关表单即可追加上传放行通知或报关单。
- 物流模块改为条件渲染：
  - 无数据时仅显示轻量空状态和“新增物流记录”按钮
  - 有数据时展示国内物流 / 国际物流两块结构化快照

### 导航与基础数据
- 左侧导航顺序调整为：
  - 控制台
  - 客户
  - 供应商/合作伙伴
  - 订单
  - 财务
  - 物流
  - AI Beta
  - 系统设置
- 新增 `供应商/合作伙伴` 模块，支持维护工厂、货代、报关行等伙伴档案。
- 客户页改为高密度表格，增加客户名 / 国家筛选，并在国家列加入国旗图标。

### 设置与编码规则
- 系统设置新增“单据编码规则”区块。
- 支持维护 `order_number_prefix`，新建订单会按配置前缀生成编号。
- 旧订单号保持不变，前缀变更仅影响后续新建订单。

### 后端与数据模型
- 新增 `partners` 表，用于统一管理供应商 / 合作伙伴。
- 新增 `production_plans` 表，用于订单级单条生产安排。
- 财务记录新增伙伴关联能力，付款录入时可直接关联伙伴并回显伙伴名称。
- 新增 `/api/settings/document`、`/api/partners`、`/api/orders/:id/production`、`/api/production/:id` 等接口。

## [2026-04-22]

### MVP 稳定化
- 完成 SQLite 增量迁移与单库收敛，统一数据库入口。
- 登录 Cookie 调整为开发 / 生产环境感知配置，本地开发可直接在 `localhost` 正常使用。
- `/api/settings/ai` 改为登录后可读写。
- 前端统一 API 错误处理，未登录时自动回到登录流程。

### 核心业务闭环
- 客户模块补充编辑能力，并禁止删除仍有关联订单的客户。
- 订单模块打通 `草稿 -> 生产中 -> 报关中 -> 发货中 -> 已完成` 主流程。
- 财务模块支持币种、分类、编辑与按订单联动。
- 物流模块支持按订单维护、编辑、同步回显。
- Dashboard 改为真实数据库聚合，不再展示纯假数据。

### 订单页一阶段重构
- 订单列表页改为“轻列表 + 强筛选”，支持关键词、国家、状态、下单月份、发货月份等筛选。
- 订单详情拆成独立页面 `/orders/:id`，不再使用同页下方展开的长详情。
- 订单详情页打通三个核心录入动作的基础骨架：
  - 编辑订单内容
  - 录入款项
  - 发货 / 更新物流

### 高密度工作台基础版
- 订单详情页引入统一右侧 Drawer 交互。
- 财务记录、物流记录、报关信息开始在订单详情页内联维护。
- 订单明细改为表格化展示，并补充运费 / 杂费 / 总计汇总。
- 新增附件上传与本地存储基础能力，支持财务 / 报关 / 物流附件。
