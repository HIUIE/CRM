# Changelog

## [2026-04-24]

### 第七轮深度打磨: 高频操作回归、物流逻辑重构与后端优化
- **高频操作路径回归**:
  - 在订单详情页顶部 Header 恢复了平铺的快捷按钮组：[编辑订单]、[录入收支]、[同步生产]、[更新报关]、[新建物流]。
  - 确保核心业务流操作“一键可见”，大幅减少层级跳转与菜单点击。
- **物流与包装逻辑彻底重构**:
  - **装箱/包装数据 (Packing Data)**: 将包装尺寸、毛重、件数从物流节点中剥离，升级为独立的动态列表模块，支持多行录入，完美适配“一单多箱、多规格包装”的业务场景。
  - **运输轨迹 (Transit Routing)**: 物流模块回归纯粹的运输管理，仅保留承运商、单号及时间轨迹，并修复了运单附件上传后不显示的逻辑 Bug。
- **生产时间轴瘦身与修复**:
  - **交互修复**: 解决了“添加进度”弹窗中进度描述文本域无法输入的 Bug。
  - **视觉优化**: 替换时间轴圆点为“时钟”图标，收紧行间距至 16px，同屏信息展现率提升 40%。
- **附件预览与存储优化**:
  - **全格式支持**: 修复了 JPG/PNG 图片点击无法预览的问题，现在图片与 PDF 均支持在当前页面直接弹窗查看。
  - **结构化归档**: 后端文件存储架构重构，上传资料将按“客户 -> 订单”的物理路径进行逻辑归类，彻底告别资料散乱。
- **细节打磨**:
  - 扩大了“内部备注”输入框体积，支持多行长文本顺畅录入。
  - 重排了订单列表的快捷时间筛选顺序：近一周、本月、上月、近 3 月、半年、近 1 年，更符合日常经营复盘直觉。

### 第六轮深度优化: 数据密度、精简工作流与字段重构
- **全局 UI 空间优化与“去冗余”**:
  - **删除全局重复标题**: 删除了客户、伙伴、订单等模块最外层的宏观标题区域，将搜索与核心操作直接整合进页面 Header。
  - **优化控制台间距**: 适当压缩了 Dashboard 指标卡片间的边距，提升了同屏数据展现率，使界面更加紧凑专业。
- **快捷筛选器 (Quick Filter Chips)**:
  - 在“客户”、“订单”、“财务”模块引入了时间维度快捷筛选：[本周], [本月], [近3个月], [近半年], [近1年], [全部]。
  - UI 严格遵循 Verdana 规范，支持瞬间切换过滤，大幅提升日常作业效率。
- **客户模块 (Customer) 字段重构**:
  - **精简字段**: 移除了原有的“付款条件”和“物流偏好”全局字段（这些应在具体订单中灵活决策）。
  - **新增高优字段**: 增加了“客户来源渠道”（如阿里巴巴、官网、展会等下拉单选）和“意向产品类型”（多行文本），完善客户画像。
- **伙伴模块 (Partner) 体验深度升级**:
  - **整行点击交互**: 伙伴列表现在支持整行点击直接进入编辑模式，无需精确点击操作按钮。
  - **新增星级评定**: 引入 1-5 星级“合作度/配合度”评定，业务员可直观评估供应商优先级。
  - **补全关键信息**: 表单新增“联系人”、“详细地址”字段，满足日常寄样与联络需求。
- **缺陷修复**:
  - **后端稳定性**: 修复了 `readString` 未定义导致的 `ReferenceError`，解决了页面因后端崩溃导致的“Failed to fetch”报错。
  - **多币种汇总**: 优化了 Dashboard 后端汇总逻辑，StatCard 现在能更准确地按币种展示（USD/CNY 分开统计）。

### 业务逻辑深化与表单闭环 (Verdana Health 规范纠偏续)
- **全局交互优化**:
  - **保存并刷新**: 修复了所有 Drawer 弹窗的保存逻辑，确保数据真实提交后端、弹窗自动关闭且父页面实时局部刷新。
  - **样式规范化**: 全局移除了正文中的斜体样式，统一使用 DM Sans 常规体，彻底摒弃“高保真、作业协议”等极客辞令。
  - **附件能力增强**: 所有附件名字现在支持点击直接预览，侧边图标支持真实下载功能。
- **模块深度改造**:
  - **订单明细**: 
    - 文案更正为“编辑订单明细”、“产品列表”、“运费与杂费”。
    - 打通了动态产品分录表格，支持名称/型号、单价、数量、单位（pcs, sets, kg, m, rolls）的实时录入与总价计算。
  - **财务信息**: 
    - 引入多币种支持，收支录入时可选择 USD, CNY, EUR, GBP, HKD 等。
    - 财务概览支持按币种切换展示回款进度。
  - **生产信息**: 
    - 修复了生产日报无法输入的 Bug，升级为“生产进度时间轴”模式。
    - 记录支持结构化日期、文字描述及多个附件关联，并在主页以美观的时间轴形式展现。
  - **报关信息**: 
    - 修复了报关表单字段串位问题（移除了物流相关字段）。
    - 新增“贸易方式”选择，优化了电子凭证列表 UI，统一 Icon 大小并去除了冗余名称显示。
  - **物流信息 (重大重构)**: 
    - 新增“国内/国际”物流模式切换，根据选择动态展示差异化字段。
    - 国内物流包含：承运公司、发货日期、件数、重量、包装尺寸、收货地址等。
    - 国际物流包含：运输方式（海运/空运/铁路/陆运）、提单号、预计时间等。
- **底层架构支持**:
  - 扩展数据库 `customs_records`、`logistics_records`、`production_logs` 表字段以支持上述新业务属性。
  - 后端 Payload 校验逻辑同步更新，支持多币种汇总与生产进度附件关联。

### 视觉与体验优化 (Verdana Health 规范纠偏)
- **视觉减负与结构清理**:
  - 删除了与顶部步骤条功能重复的“关键节点核对”模块。
  - 右侧导航标题从斜体英文 `SECTION_NAV` 回归标准中文加粗 **“页面导航”**。
  - 附件列表移除违和的 `SECURED`（协议加密）小字标签。
  - 优化“订单明细”合计行背景色，由重黑色改为浅灰色 (`#F1F5F9`)，提升视觉呼吸感。
  - 压缩右下角 AI 辅助卡片体积，由黑色大色块改为标准 Primary Navy 按钮。
  - 减弱次要操作按钮视觉权重，将卡片头部按钮改为 `LightActionButton` (线框样式)，仅保留核心操作为实心填充。
- **文案“降维”规范 (去极客化)**:
  - 全面清理“军事化、极客化”词汇，回归直白 B2B 商业术语：
    - “订单货物明细” -> **“订单明细”**
    - “内部业务备注指令” -> **“内部备注”**
    - “财务流水审计” -> **“财务信息”**
    - “财务交易流水” -> **“收付款记录”**
    - “资产变动” -> **“金额”**
    - “生产排产协议” -> **“生产信息”**
    - “报关资料档案” -> **“报关信息”**
    - “货运物流实时快照” -> **“物流信息”**
  - 替换 `NULL_MANIFEST`、`REPOSITORY_EMPTY`、`LOAD_FAILURE` 等技术占位符为自然中文文案。
- **缺陷修复**:
  - **乱码清理**: 彻底清除报关附件中的 `Æ Å È` 等乱码，同步更新 `seed:demo` 脚本生成真实的中文文件名（如：商业发票_2026.pdf）。
  - **稳定性恢复**: 修复了因重构导致的 `AttachmentEditor` 组件丢失及 Handler 函数引用错误导致的页面无法打开问题。

### 生产上线加固
- 新增生产启动脚本 `npm run start`，以 `NODE_ENV=production` 启动并服务 `dist/` 静态资源。
- 新增一键上线检查 `npm run release:check`，依次执行：类型检查、测试、构建、UI smoke。
- 服务启动日志补齐生产排查信息：运行模式、本机/局域网访问提示、数据库路径、上传目录。
- 启动失败信息更明确（如端口占用、无权限监听端口等）。

### 局域网登录与安全初始化
- Cookie 策略改为局域网 HTTP 友好：默认不强制 `secure`，仅 `COOKIE_SECURE=true` 才启用 HTTPS-only Cookie。
- 生产环境强制要求配置 `JWT_SECRET`，避免继续使用预览默认密钥。
- 新库首次初始化管理员 `root`：生产环境必须设置 `INITIAL_ADMIN_PASSWORD`，否则启动会直接失败。

### 健康检查与路径可配置
- `/api/health` 扩展返回：`dbPath`、`uploadsPath`、`uploadsUrl`、`timestamp`，便于生产排查。
- 上传目录支持环境变量 `UPLOADS_DIR`，避免 smoke/测试数据写入正式附件目录。

### Smoke 与备份
- 新增 `npm run smoke:ui`：使用临时数据库与演示数据进行生产模式 smoke（有 Playwright 则跑浏览器，无则降级为 HTTP/API 检查）。
- 新增 `npm run backup`：将 SQLite 数据库与 `uploads/` 备份到按时间戳命名的目录（可用 `BACKUP_DIR` 指定位置）。

### 文档
- `.env.example` 更新为局域网生产可用模板，补齐 `HOST/PORT/CRM_DB_PATH/UPLOADS_DIR/JWT_SECRET/INITIAL_ADMIN_PASSWORD/COOKIE_SECURE`。
- README 补充“第一版生产部署步骤”、上线前检查、备份与恢复流程。

## [2026-04-23]

### 团队稳定化
- 后端路由继续按领域收口，统一保留标准错误结构：`error.code` + `error.message`。
- 新增基础用户管理接口，支持创建账号、启停用户、重置密码。
- 权限边界进一步收紧：
  - `admin` 管理系统设置、AI 配置、团队账号和关键删除动作

- 登录页和系统设置页文案改为正式团队入口语气，不再展示默认管理员账号。

- 主要列表页补充“创建人 / 最近操作人”回显，方便团队协作场景下追踪记录来源。
- AI 页面文案收敛为“订单草稿解析 + 风险诊断”两类辅助用途，不再强调测试态限制。

- README 重写为“局域网小团队可用”版本，新增：
  - `/api/health`
  - `CRM_DB_PATH`
  - `uploads/`
  - 局域网部署建议
  - SQLite 与附件目录备份说明

### AI
- AI 解析入口不再被硬编码限制为 Gemini。
- `/api/ai/parse-order` 现在会根据系统设置中的模型自动分流：
  - `Gemini` 继续走 Google GenAI SDK
  - `deepseek-chat` 改走 OpenAI 兼容接口
- 新增对 `ai_base_url` 的兼容读取；当模型为 DeepSeek 且未手填 Base URL 时，后端会优先使用默认官方兼容地址。
- AI 页面去掉“仅支持 Gemini”的前端限制提示，只要配置了可用 Key 即可发起解析。

### 订单工作台
- 订单详情页继续重构为高密度工作台，形成稳定的模块顺序：
  - 基础信息
  - 订单明细
  - 生产安排
  - 财务流转
  - 报关信息
  - 物流信息
- 顶部阶段条支持锚点跳转到对应模块。
- 顶部财务概览支持 `待收金额 = 0` 时显示 `已结清` 状态。
- 基础信息区域移除了 `关键里程碑` 与 `产品摘要` 的主展示。
- 新增 `生产安排` 模块与 Drawer，支持选择工厂/供应商、维护生产状态与验货状态。
- 报关模块支持“后补附件”，无需重新打开完整报关表单即可追加上传放行通知或报关单。
- 物流模块改为条件渲染：
  - 无数据时仅显示轻量空状态和“新增物流记录”按钮
