# Changelog

## [2026-04-24]

### 第十八轮深度优化: 贴合真实外贸业务流与 UI 细节微调
- **物流业务逻辑重构 (Logistics Business Logic)**:
  - **货代与承运商分离**: 物流模块现在严格区分“货运代理 (Freight Forwarder)”与“实际承运商 (Actual Carrier)”，符合外贸企业通过中介货代发货并向其支付费用的真实对账场景。
  - **字段精准化**: 每一段运输记录现在都包含独立的货代名称、承运商名称、提单号/运单号及发货时间，提升了物流追踪的专业维度。
- **财务与生产模块 UI 优化**:
  - **快捷录入回归**: 在财务信息卡片右上角筛选 Tab 旁恢复了 [+ 录入收支] 快捷按钮，缩短了业务员日常记账的操作路径。
  - **界面降噪**: 删除了生产信息卡片中订单号右侧冗余的 [+] 图标，保持控制台视觉上的绝对极简。
- **稳定性与逻辑闭环**:
  - 修复了后端物流保存接口，确保新字段 `freight_forwarder` 能够正确写入数据库并实现跨模块读取。

### 第十七轮优化: 订单单据编码 (流水号) 规则重构
- **全新编码规则定义**:
  - 废弃了旧的静态自增逻辑，改为符合外贸实战习惯的 **“日期型每日重置流水号”**。
  - 标准格式: `CQBX-YYYY-MMDDXX` (例如: `CQBX-2026-042401`)。
  - 规则包含: 固定业务前缀 `CQBX`、四位年份、月日拼接位、以及每日从 `01` 开始重置的两位数流水号。
- **后端防并发实现**:
  - 重构了订单创建 API。系统现在会自动查询当天已生成的最大序号，并智能 +1。
  - 引入了数据库 **Transaction (事务)** 机制，确保在多人同时点击创建订单时，单据编号的唯一性与连续性，彻底杜绝重号现象。
- **系统设置同步**:
  - 同步更新了“系统配置”中的编号规则预览，使管理员能直观感知最新的编码逻辑。

### 第十六轮安全加固: 数据导出、附件鉴权与敏感路径防泄漏
- **管理员专属全量导出**:
  - 新增 `GET /api/settings/export?format=zip-csv` 管理员导出接口，支持将客户、订单、财务、物流、报关、生产、附件等核心业务数据打包为 **ZIP + 多 CSV** 下载。
  - 前端系统设置页新增“导出全量数据”按钮，并根据角色限制仅对 `admin` 可见。
- **附件读取链路重构**:
  - 废弃 `uploads/` 静态公开映射，所有附件预览与下载统一切换为受登录态保护的 `GET /api/files/:id/:storedName`。
  - 新增文件名安全校验、路径归一化与根目录 containment 检查，阻断 `../` 等路径穿越攻击。
  - 统一清洗历史附件路径兼容逻辑，兼容旧 `uploads/...` 记录与新相对路径记录。
- **Express 安全面加固**:
  - 接入 `Helmet`，关闭默认 `X-Powered-By` 暴露，补齐基础安全响应头。
  - 在静态资源与 SPA 路由前增加敏感路径拦截，默认拒绝 `/.env`、`/.git/*`、`*.sqlite`、`*.db`、`*.pem`、`*.key` 以及旧 `/uploads/*` 探测请求。
  - 收紧 `GET /api/health` 返回内容，不再暴露数据库与上传目录的物理路径。
- **部署约束与底层补强**:
  - 新增生产启动校验，若 `CRM_DB_PATH` 或 `UPLOADS_DIR` 落在 `dist/`、`public/` 静态目录内则直接拒绝启动。
  - 补齐 `packing_records` 表迁移，消除订单详情与装箱附件相关的隐藏 schema 缺口。
  - 测试覆盖新增导出权限、文件鉴权、路径穿越、敏感路径拦截与生产目录校验。

### 第十五轮优化: 全局提示框 (Toast) 体验升级
- **自动消失时长优化**:
  - 全局成功/信息类提示框的停留时间统一优化为 **3000ms (3秒)**，解决了之前停留过长遮挡操作视线的痛点。
- **增强交互体验**:
  - **手动快速关闭**: 现在提示框支持点击即刻消失，并新增了显式的“关闭 (×)”按钮，给予用户更高的控制权。
  - **悬浮暂停计时**: 引入了“鼠标悬停暂停”逻辑。当用户将鼠标移至提示框上时，消失倒计时将自动暂停；移开后恢复，方便用户仔细阅读长提示。
- **组件底层加固**:
  - 重构了 `OrderDetail` 中的提示逻辑，支持更优雅的计时器管理，避免内存泄漏。

### 第十四轮修复: 主标题数据绑定乌龙 (Critical Data Binding Error)
- **正名“核心客户”**: 将详情页主标题从错误绑定的 ID 重新指向 **[客户名称 / 公司名称]**，确保业务逻辑闭环。
- **视觉减负**: 彻底清理了商品明细卡片底部残留的旧版备注行，使列表回归纯净。

### 第十三轮核心逻辑修复: 数据跨模块联动与算数 Bug
- **算术 Bug 彻底修复**: 修正了“商品明细”中的合计逻辑，由静态读取改为实时动态累加 `∑(单价 × 数量) + 运杂费`。
- **跨模块数据绑定**: 财务进度条现在直接同步“商品明细”中的实时合计总值，彻底杜绝了数据断层。
- **进度条溢出逻辑**: 优化了已收金额超过订单金额时的进度条展示（显示为 100% 满格）。

### 第十二轮 UI 像素级纠偏: 顶部对齐与黑边修复
- **像素级对齐**: 修复了页面导航与主内容区顶部不对齐的问题，确保视觉起始线绝对水平。

### 第十一轮深度打磨: 数据流闭环、彻底汉化与排版纠偏
- **核心数据流 Bug 修复**:
  - **订单状态同步**: 修复了“编辑订单”弹窗中修改状态无法保存的 Bug。现在状态修改已实现前后端双向绑定，并能实时驱动顶部步骤条（Stepper）更新。
  - **产品明细即时渲染**: 解决了新增/编辑产品后列表不刷新的问题。通过后端 `PATCH` 逻辑重构，现在产品明细能与订单主表同步保存并立即呈现。

- **布局与排版逻辑优化**:

- **全站彻底汉化 (Complete Localization)**:
  - 进行了严格的全局文案清洗，彻底消灭描述性英文词汇：
    - `INLAND` -> **国内运输**

  - **质检节点明确化**: 优化了生产信息中质检状态的交互提示，增加了下拉箭头引导，明确其可交互性。
